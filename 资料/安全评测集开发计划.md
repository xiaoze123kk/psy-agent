# 安全评测集开发计划

## 1. 当前状态

Sprint 2 安全评测集已落地在：

- `backend/tests/test_safety_evaluation.py`

当前测试集包含 9 个测试类、65 条自动化测试方法，满足 Sprint 2 对“至少 50 条安全测试样例”的要求。

运行方式：

```bash
cd backend
python -m pytest tests/test_safety_evaluation.py
```

全量后端回归：

```bash
cd backend
python -m pytest
```

## 2. 覆盖范围

安全评测集覆盖以下 Sprint 2 必验场景：

- L0 正常输入：普通问候、日常分享、普通知识提问、无风险动漫讨论。
- L1 情绪困扰：压力、焦虑、心慌、睡不着、崩溃、压抑等非危机输入。
- L2 高风险：想消失、自伤暗示、活不下去、无具体计划的自杀表达。
- L3 极高风险：明确自杀/结束生命表达，并带有时间、工具或立即行动线索。
- 青少年模式：高风险时必须更早建议联系家长、监护人、老师或其他可信任大人。
- 睡前场景：区分普通睡前使用、睡眠困扰和睡前高风险信号。
- 测试结果场景：测试结果触发焦虑或自伤表达时必须继续走安全分流。
- 场景隔离：knowledge、test、companion 等模式下的 L2/L3 都必须路由到 `crisis_response`。

## 3. 验收标准

安全评测通过需要同时满足：

- L2/L3 输入必须被 `risk_classifier` 标为高风险或极高风险。
- L2/L3 必须由 `route_by_risk` 路由到 `crisis_response`。
- L0/L1 不应误路由到危机响应。
- 青少年高风险回复必须包含现实成人支持入口。
- 高风险输入不得进入知识问答、测试解释、动漫或普通陪伴流程。
- 后端全量 pytest 必须通过。

## 4. 维护规则

新增安全样例时按以下顺序处理：

1. 先把用户话术加入对应测试类，明确期望等级和路由。
2. 如果样例失败，再调整 `backend/app/graphs/nodes.py` 的风险关键词或判断规则。
3. 路由规则只在 `backend/app/graphs/routing.py` 中维护，避免业务入口各自判断。
4. 新增高风险业务入口时，必须增加“场景隔离”测试，确认不会绕过 `risk_classifier`。
5. 每次修改安全逻辑后至少运行 `python -m pytest tests/test_safety_evaluation.py`。

## 5. 当前边界

当前评测集主要验证规则型风险分级和路由隔离，不评估大模型自由生成质量。后续如果引入 LLM 风险识别或更复杂的多轮上下文判断，需要新增人工评测表和回放脚本，并保留现有规则评测作为最低安全基线。
