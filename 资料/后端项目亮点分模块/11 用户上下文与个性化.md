# 用户上下文与个性化

> 返回：[[00 后端项目亮点分模块索引]]
> 相关代码：`backend/app/services/user_context_service.py`、`backend/app/services/user_context_pack_service.py`、`backend/app/services/safety_context_service.py`、`backend/app/services/companion_style.py`

## 它解决什么问题

同一句话，对不同用户应该有不同回应方式。后端会在每轮对话前构造上下文包。

## 上下文包括什么

- 用户 profile digest。
- 用户目标状态 goal_state。
- session digest。
- retrieved memories。
- safety context pack。
- companion style。
- teen/adult mode。
- temporal context。

这些会进入 `AgentState`，再被 prompt builder 和不同节点使用。

## 个性化不是全量历史

这个项目没有把所有历史原样塞进 prompt，而是经过整理：

```text
用户资料 / 会话摘要 / 长期记忆
  -> digest / goal_state / context_pack
  -> AgentState
  -> prompt builder
```

这样既能保留连续性，也能减少上下文污染。

## 为什么是亮点

它让 agent 的“个性化”不是直接把所有历史塞进 prompt，而是经过整理后的 digest/context pack。这样更可控，也更容易审计。

## 开发边界

- 新增个性化字段时，要检查 AgentState、prompt builder、trace summary。
- 青少年模式相关逻辑要同步检查风险和回复按钮。
- safety context pack 不应和普通用户上下文混淆。
