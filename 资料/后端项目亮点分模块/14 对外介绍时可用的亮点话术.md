# 对外介绍时可用的亮点话术

> 返回：[[00 后端项目亮点分模块索引]]
> 用途：面试、答辩、项目展示、README 摘要。

## 风控亮点

> 我们把心理风险控制做成了 LangGraph 早期节点。风险等级会影响路由、RAG、记忆、工具和 prompt contract。高风险不会进入普通聊天或咨询语料 RAG，而是走危机安全路径。

## RAG 亮点

> RAG 不只是知识检索，还包括咨询语料过程检索。语料按 turn_pair、process_segment、session_sketch 分层，支持 Milvus 召回、本地 reranker、配额选择、失败降级和 trace。

## 记忆亮点

> 记忆系统按类型、可见性、风险等级和用户记忆模式控制读写。检索时结合规则分、向量分、重要性、新鲜度和目标相关性；写入通过候选提取和后台 job，避免主链路阻塞。

## 校验亮点

> 回复前有安全和体验双校验。除了危险内容，还检查问题过多、过度心理化、忽略用户锚点、文化作品胡编、重复模板结构等心理陪伴产品特有问题，失败时可触发重写。

## 可观测性亮点

> 每轮对话保存 graph trace、RAG trace、validator reasons、quality trace 和 memory decisions。这样可以解释一次回复为什么这样生成，也方便定位策略链路中的问题。

## 对话体验亮点

> conversation move policy 独立控制当前轮的对话动作、声线、问题预算、分析深度和结构变化。它能根据用户“别分析我”“别一直问”等纠偏信号做短期适配，减少模板感。

## 一分钟版本

> 这个项目是一个面向心理咨询陪伴的 LangGraph agent 后端。它不是简单调用大模型，而是把一轮对话拆成风险识别、控制面路由、咨询语料 RAG、长期记忆、场景化回复、内容校验和 trace 追踪。高风险场景会优先走安全路径，RAG 和记忆会受风险策略控制，输出前还会做安全和体验双校验。整体目标是让 agent 同时具备安全性、连续性、可解释性和更自然的对话体验。

## 两分钟版本

> 后端最核心的是一张 LangGraph 主图：用户输入先经过 normalize 和用户画像加载，然后做风险识别和 control plane 分流。高风险、临床红旗、边界或澄清场景会走专门回复节点；普通支持才进入意图识别和咨询语料 RAG。回复生成后，还要经过 response validator，检查危险内容、过度心理化、追问过多、忽略用户锚点、RAG 复制等问题。最后系统会摘要本轮、提取记忆候选，并按用户记忆模式和风险策略决定是否写入长期记忆。
>
> 项目的亮点是这些模块之间不是孤立的。风险等级会影响 RAG、记忆、工具和 prompt contract；conversation move policy 会影响 prompt 和 validator；Graph trace 和 quality trace 会记录每轮对话为什么这么走。这让它更像一个工程化的心理陪伴 agent，而不是普通聊天机器人。
