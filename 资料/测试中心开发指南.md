## 3. 核心架构概念

```
test_type（分类，固定3种）      test_id（测试，每类可多个）
─────────────────────────      ───────────────────────────
state                        → state-check-v1（今日状态）、night-state-v1（未来）
personality                  → mbti-sixteen-type（MBTI 16型人格·混合题库）、big-five-v1（未来）
anime                        → anime-match-v1（动漫角色，draft）
```

- **`test_type`** = 分类维度。后端、前端路由均以此字段分发评分和结果页模板
- **`test_id`** = 测试唯一 ID。用于题库文件查找、attempt 关联
- **`test_code`** = 英文代码。保留字段，可为空
- **`status`** = `"published"`（可测试）或 `"draft"`（不可测试）

### 3.1 架构变革（v2）

从 v2 开始，测试中心采用**题目数据 + 评分算法 + 代码完全解耦**的架构：

| 组件 | 位置 | 职责 |
|------|------|------|
| `*title*.json` | `data/tests/{type}/{test_id}/` | 题目元数据（题目文本、选项） |
| `*score*.json` | `data/tests/{type}/{test_id}/` | 评分规则（选项分值、阈值、结果描述） |
| `scorer.py` | `data/tests/{type}/{test_id}/` | 评分算法模块（按套独立，互不干扰） |
| `test_service.py` | `app/services/` | 编排层（加载 → 动态 import scorer → 调用 compute → 写数据库） |

每套测试独立文件夹，修改评分算法只需改对应 `scorer.py`，无需动核心代码。新增测试只需创建新文件夹 + 三件套，零侵入。

## 5. API 接口列表

| 方法 | 路径 | 认证 | 说明 |
|---|---|---|---|
| `GET` | `/api/v1/tests` | 否 | 测试列表（含 draft） |

| `GET` | `/api/v1/tests/history` | 是 | 用户测试历史列表 |

## 6. 新增测试（在已有分类下，不改变分类）

**适用场景**：在"状态"下新增"夜间状态"，或在"人格"下新增"大五人格"，或在"动漫"下新增"反派测试"。

### 6.1 新架构下的流程（v2 推荐）

无需修改任何 Python 代码，只需三步：

```bash
# 1. 创建测试文件夹
mkdir -p data/tests/state/night-state-v1/

# 2. 放入题目文件 title.json
# 3. 放入评分规则文件 score.json
# 4. 放入评分算法 scorer.py
```

> **重要**：新增测试无需修改 `test_service.py`，`_load_all_tests()` 会通过 `*title*.json` glob 自动发现新测试。

### 6.2 title.json 格式（必需）

所有测试的 title JSON **统一采用以下格式**，每题必须包含 `id`（稳定标识）、`text`、`options`：

```json
{
  "test_id": "night-state-v1",
  "code": "night_state",
  "title": "夜间状态评估",
  "test_type": "state",
  "estimated_minutes": 3,
  "audience": "all",
  "status": "draft",
  "questions": [
    {
      "id": "night-q-0",
      "index": 0,
      "text": "最近一周睡前你通常在想什么？",
      "options": [
        {"id": "A", "text": "回顾白天的事情"},
        {"id": "B", "text": "操心明天要做的事"},
        {"id": "C", "text": "比较放松，不特意想什么"},
        {"id": "D", "text": "反复担心各种可能的问题"}
      ]
    }
  ]
}
```

**questions 元素字段**：

| 字段 | 说明 |
|------|------|
| `id` | **稳定唯一标识**（如 `"mbti-q-000"`、`"state-q-0"`），题目打乱后此 ID 不变，用于关联答案 |
| `index` | 当前题序（从 0 开始），打乱后改变 |
| `text` | 题目正文（**所有题型均需提供**） |
| `options` | 选项列表，每项含 `id`（如 `"A"`）和 `text` |
| `metadata` | 可选，附加数据（如 MBTI 的 `content` 富文本），scorer.py 自行读取 |

> 选项的计分值**不在 title.json 中**，统一放在 score.json 的 `scoring` 字段。

### 6.3 score.json 格式（统一）

所有测试的 score JSON **统一采用以下两层结构**：

```json
{
  "test_id": "night-state-v1",
  "scoring_method": "sum",
  "results": {
    "good-sleep": {
      "result_code": "good-sleep",
      "result_label": "睡眠状态良好",
      "summary": "...",
      "suggested_actions": ["..."],
      "risk_note": "...",
      "traits": ["..."],
      "strengths": ["..."],
      "blind_spots": ["..."],
      "companion_style": "..."
    }
  },
  "default_result": { ... },
  "scoring": {
    "option_scores": { "0": {"A": 3, "B": 2, "C": 4, "D": 1} },
    "thresholds": [
      {"min_total": 10, "result_code": "good-sleep", ...}
    ]
  }
}
```

**顶层字段**：

| 字段 | 说明 |
|------|------|
| `test_id` | 测试 ID |
| `scoring_method` | 评分方法描述 |
| `results` | **结果解释映射**，统一字段包括 `result_code`、`result_label`、`summary`、`suggested_actions`、`traits`、`strengths`、`blind_spots`、`companion_style` |
| `default_result` | 可选兜底结果 |
| `scoring` | **评分规则**（测试特定格式），scorer.py 从本字段读取具体计分逻辑 |

MBTI 等复杂测试的 `scoring` 可包含 `traditional_dimensions`、`cognitive_stacks`、`cross_validation_rules` 等自定义字段。

### 6.4 scorer.py 接口契约

```python
def compute(test: dict, answers: dict[int, str]) -> dict:
    """评分函数。

    参数:
        test: 来自 title.json 的完整测试 dict（含 questions）
        answers: {question_index: option_id}，如 {0: "A", 1: "B", ...}

    返回 dict 必须包含以下字段:
        result_code, result_label, summary, suggested_actions
        traits, strengths, blind_spots, companion_style

    可选字段（personality 类型建议提供）:
        sixteen_type_code, sixteen_type_label
    """
    ...
```

### 6.5 加载机制说明

- `test_service.py` 通过 `importlib.import_module(f"data.tests.{type}.{test_id}.scorer")` 动态加载评分模块
- 每个测试的 scorer.py 是独立 Python 模块，内部可 import 标准库或同目录 JSON
- scorer 缺失时 `complete_attempt()` 返回 None，不影响其他测试，错误记录到 logger

### 6.6 前端：无需修改

如果新测试的 `test_type` 属于已有的三类之一，结果页模板**完全复用**。

如果新测试的结果页需要不同布局（极少见），在 `App.vue` 结果页区块新增 `v-else-if` 分支。

### 6.7 前端：演示模式

在 `App.vue` 的 `demoTests` 列表追加新测试卡片数据，并在 `demoXXXResult` 新增对应的演示结果数据（如果有的话）。
