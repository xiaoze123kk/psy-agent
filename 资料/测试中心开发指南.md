## 3. 核心架构概念

```
test_type（分类，固定3种）      test_id（测试，每类可多个）
─────────────────────────      ───────────────────────────
state                        → state-check-v1（今日状态）、night-state-v1（未来）
personality                  → sixteen-type-v1（16型人格）、big-five-v1（未来）
anime                        → anime-match-v1（动漫角色，draft）
```

- **`test_type`** = 分类维度。后端、前端路由均以此字段分发评分和结果页模板
- **`test_id`** = 测试唯一 ID。用于 `_TESTS` 查找、attempt 关联
- **`test_code`** = 英文代码。保留字段，可为空
- **`status`** = `"published"`（可测试）或 `"draft"`（不可测试）

## 4. API 接口列表

| 方法 | 路径 | 认证 | 说明 |
|---|---|---|---|
| `GET` | `/api/v1/tests` | 否 | 测试列表（含 draft） |
| `GET` | `/api/v1/tests/{test_id}` | 否 | 测试详情（题目+选项） |
| `POST` | `/api/v1/tests/{test_id}/attempts` | 是 | 开始测试，创建 attempt |
| `POST` | `/api/v1/tests/attempts/{attempt_id}/answers` | 是 | 提交单题答案 |
| `POST` | `/api/v1/tests/attempts/{attempt_id}/complete` | 是 | 完成测试，获取结果 |
| `GET` | `/api/v1/tests/attempts/{attempt_id}/result` | 是 | 查询已完成 attempt 的结果 |
| `GET` | `/api/v1/tests/history` | 是 | 用户测试历史列表 |

## 6. 新增测试（在已有分类下，不改变分类）

**适用场景**：在"状态"下新增"夜间状态"，或在"人格"下新增"大五人格"，或在"动漫"下新增"反派测试"。

### 6.1 后端：添加种子数据

编辑 `backend/app/services/test_service.py`，在 `_TESTS` 列表末尾追加一条（以 state 分类为例）：

```python
{
    "test_id": "night-state-v1",        # 唯一
    "code": "night_state",              # 唯一，英文下划线
    "title": "夜间状态评估",
    "test_type": "state",               # state / personality / anime
    "estimated_minutes": 3,
    "audience": "all",
    "status": "draft",                  # 先 draft，题目就绪后改为 published
    "questions": [
        {
            "index": 0,
            "text": "最近一周睡前你通常在想什么？",
            "options": [
                {"id": "A", "text": "回顾白天的事情", "score": 3},
                {"id": "B", "text": "操心明天要做的事", "score": 2},
                {"id": "C", "text": "比较放松，不特意想什么", "score": 4},
                {"id": "D", "text": "反复担心各种可能的问题", "score": 1},
            ],
        },
        # ... 更多题目
    ],
},
```

> **重要**：`test_type` 必须填写为 `"state"` / `"personality"` / `"anime"`。
> 同一 `test_type` 下的所有测试共享评分逻辑和结果页模板，无需改路由代码。

### 6.2 后端：评分逻辑（如需独立算法）

如果新测试的评分方式与同分类已有测试**不同**，需要写独立函数。编辑 `complete_attempt` 中的路由分支（约 L403-407）：

```python
# 当前 state 分类的评分由 _compute_state_result 处理
# 如果"夜间状态"需要不同的计算方式，将路由由 test_type 改为 test_id：

if t["test_id"] == "night-state-v1":
    result_data = _compute_night_state_result(t, attempt.answers)
elif t["test_type"] == "personality":
    result_data = _compute_sixteen_type_result(t, attempt.answers)
elif t["test_type"] == "anime":
    result_data = _compute_anime_result(t, attempt.answers)
else:
    result_data = _compute_state_result(t, attempt.answers)
```

评分函数必须返回标准 dict：

```python
def _compute_night_state_result(test, answers) -> dict:
    return {
        "result_code": "healthy-sleep",
        "result_label": "睡眠状态良好",
        "summary": "...",
        "suggested_actions": ["..."],
        "profile": TestResultProfile(
            traits=["..."],
            strengths=["..."],
            blind_spots=["..."],
            companion_style="...",
        ),
    }
```

### 6.3 前端：无需修改

如果新测试的 `test_type` 属于已有的三类之一，结果页模板**完全复用**。

如果新测试的结果页需要不同布局（极少见），在 `App.vue` 结果页区块新增 `v-else-if` 分支。

### 6.4 前端：演示模式

在 `App.vue` 的 `demoTests` 列表追加新测试卡片数据，并在 `demoXXXResult` 新增对应的演示结果数据（如果有的话）。
